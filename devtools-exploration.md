# DevTools Exploration

## Websites Inspected

### GitHub
- The page uses a dense but well-structured navigation system that adapts to smaller screens.
- Inspecting the DOM shows clear landmark separation and reusable components.
- DevTools is useful here for checking responsive menus, focus states, and network requests.

### MDN Web Docs
- MDN prioritizes content and search, which makes the page layout feel efficient and utility-driven.
- Semantic headings and article-style content improve readability and accessibility.
- DevTools helps reveal how the page balances readability with a large amount of documentation.

### W3C
- The W3C site uses a straightforward information hierarchy with a content-first layout.
- The design is less decorative and more documentation-focused, which keeps navigation clear.
- DevTools highlights simple structure, strong text contrast, and predictable spacing.

## Key Takeaways
- A good page usually has clear landmarks, meaningful headings, and responsive behavior.
- DevTools is most useful when checking layout shifts, accessibility, and how components behave across screen sizes.
- The strongest sites do not rely on visual flair alone; they use structure and consistency to reduce friction.
